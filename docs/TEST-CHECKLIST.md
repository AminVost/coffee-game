# چک‌لیست تست

- درخواست OTP باعث ایجاد User با وضعیت PENDING شود.
- OTP در Browser یا Network Response نمایش داده نشود.
- OTP نادرست شمارنده تلاش را افزایش دهد.
- OTP صحیح حساب را ACTIVE و Mobile Verified کند.
- ثبت‌نام‌های مهمان قبلی با همان موبایل به حساب متصل شوند.
- Hold بعد از ۱۵ دقیقه از ظرفیت خارج شود.
- پرداخت PENDING در پنل قابل تأیید/اصلاح/رد باشد.
- پرداخت NEEDS_CORRECTION دوباره قابل تأیید یا رد باشد.
- داشبوردها، اخبار، گالری، تماس، رنکینگ و قرعه از MySQL خوانده شوند.
- لینک پیگیری فقط اطلاعات Mask‌شده را نمایش دهد.
- در Production، Database OTP و AUTH_SECRET نامعتبر مسدود شوند.
